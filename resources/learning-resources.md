# Open Source Contribution Learning Resources

## Beginner-Friendly Guides & Tutorials
- **GitHub: First OSS contribution guide** – Step‑by‑step instructions on making your first open source contribution. [Make your first open source contribution guide (GitHub)](https://github.com/readme/guides/first-oss-contribution)  
- **GitHub Docs: Finding ways to contribute** – How to choose projects, find issues, and contribute code or non‑code. [Finding ways to contribute to open source on GitHub](https://docs.github.com/en/get-started/exploring-projects-on-github/finding-ways-to-contribute-to-open-source-on-github)
- **Open Source Guides: How to Contribute** – Comprehensive guide covering why and how to contribute to open source projects. [How to Contribute to Open Source](https://opensource.guide/how-to-contribute/) 

## Tools & Interactive Learning
- **First Contributions** – A beginner‑friendly project that walks you through your first PR and shows projects with open issues you can try. [Make your first open source contribution in 5 minutes](https://firstcontributions.github.io/)

- **GitHub Education “Intro to Open Source” Experience** – Course and exercises on finding projects, understanding workflows, and community basics. [Intro to Open Source](https://education.github.com/experiences/intro_to_open_source)

## Project Discovery & Issue Finding
- **GitHub search filters for issues** – Learn how to use search queries like `label:"good first issue"` to find welcoming tasks. [Filtering and searching issues and pull requests](https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/filtering-and-searching-issues-and-pull-requests)
- **Beginner‑friendly OSS project lists** – Many open source repositories use “good first issue” or “help wanted” labels to make it easier to start. [good first issue](https://goodfirstissue.dev/)

## Tips for Using These Resources
- Start by going through the **GitHub first OSS contribution guide** to understand the workflow.  
- Use **First Contributions** to actually experience a real PR merge.  
- Explore **project lists** using GitHub search labels to find work you can start with.  
- Read one or two comprehensive articles to deepen your understanding of common pitfalls and etiquette.
