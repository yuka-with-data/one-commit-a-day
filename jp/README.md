# 毎日1コミットチャレンジ：小さく積み重ねて、出し続けるプロジェクト

２０２６年４月から、私は「毎日1コミットチャレンジ」を始めました。
これは単なるコードを書く習慣ではなく、自分の時間や知識を、少しずつ外に出し続けるための取り組みです。

このチャレンジの価値は、オープンソースの世界に限ったものではありません。
毎日少しでも何かを生み出し、外に出すことは、技術だけでなく、情報発信や日常のあらゆる場面につながっていきます。

ただ、「貢献」という言葉には、少し引っかかりを感じることもあります。
どこか表面的で、「いいことをしている自分」を見せるようなニュアンスで受け取られてしまうこともあるからです。

でも、本来の貢献はもっとシンプルで広いものだと思っています。
誰かの役に立とうと構えることではなく、自分ができることを、外に向けて積み重ねていくこと。それだけで十分に価値があります。

特別なスキルや大きな時間は必要ありません。大切なのは、習慣です。
小さな行動でも、それが続くことで、確実に誰かに届き、社会の一部になっていきます。

そしてそれは巡り巡って、自分のスキルアップとして返ってきます。
結局は、自分のためにもなっているのだと信じています。

## 🧪 私が実際に取り組んでいる事

この考えを形にするために、GitHub上で「毎日1コミットチャレンジ」を実践しています。

目標はシンプルで、
1年間、毎日1回は何かしらのアウトプットをコミットすること。

ここでいう「コミット」は、コードに限りません。

- コードを書く
- ドキュメントを改善する
- Issueを出す / コメントする
- 他の人のPRをレビューやコメントをする
- 学んだことを記録する

プロジェクトやオープンソースプラットフォームに対して、何か一つでも前に進める行動をすることを意味しています。

ここで重要なのは、質よりも「継続」です。

## 📜 コミットのレベル設計

コミットの内容は、毎日続けるために、無理のないレベル設計をしています。

### Level 1
- Issueへのコメント
- PRへのレビュー、コメント
- typo修正などの小さな改善
### Level 2
- Issueの作成
- PR（バグ修正・ドキュメント改善）
- バグの再現・記録
- レポジトリーの初期化、作成
### Level 3
- 新機能の実装
- リファクタリング
- チュートリアル作成
### 👉 ルールはシンプルです：
- 毎日：最低Level 1
- 週に1〜2回：Level 2
- 月に1〜2回：Level 3

## 🧠 継続するためのシステム
1日30〜60分でできるシンプルなループにしています。

1. GitHubを開く
2. やることを1つ決める
3. 小さく実行する
4. 必ず記録する

これだけです。

完璧にやる必要はありません。
継続する事の方が、オープンソースにとっては重要です。

## 🪫 フォールバック（どうしても難しい日）
どうしても忙しくて時間に余裕がない日や旅行中の場合、無理をしないための最低ラインを用意しています。
- IssueやPRを読んで、一言コメントを残す
- コードを読んで、気づきをメモする
- 気になったリポジトリをブックマークする

👉 どんなに小さくてもいいから「記録すること」は必須です

## 🔥思考の転換
「貢献」という言葉から、多くの人はこう考えがちです。

>「自分はまだ初心者だから、貢献できることがない」

>「大きなプロジェクトに参加するのは怖い」

>「自分のアウトプットが誰かの役に立つか不安」

>「価値のある大きなことをしなければいけない」

でも実際は違います。

> 小さくてもいいから、アウトプットし続ける事が価値になる」

オープンソースでは、
- 小さな改善も歓迎される
- 継続が信頼になる
- 誰かの役に立つことは、必ずしも大きなことではない
- 自分の成長にもつながる
- 関わり続けることで関係ができる

という世界です。

## 🌱 どんな人でも始められる

プログラマーや開発者、オープンソースでスキルアップを目指している方、
これからテクノロジー分野に挑戦したい方、
あるいはテクノロジーに関わりがなくても、日々の積み重ねを自分の成長につなげたい方――

**どんな方でも、今日から始めることができます。**

これを読んで、始めてみたいと思った方や、何から始めればいいか迷っている方は、GitHubのIssueから気軽に声をかけてください。
